# Ritim 0.7.2

Android uygulamasının yeni APK kurulduktan sonra kendisini hâlâ 0.7.0 olarak görmesine ve sürekli güncelleme istemesine neden olan paket sürümü hatası düzeltildi.

Fixed the Android package version mismatch that made a newly installed APK report itself as 0.7.0 and repeatedly request the same update.

## İndir / Download

- **Windows:** `Ritim-Setup-0.7.2.exe`
- **Android:** `Ritim-Android-v0.7.2.apk`

## Bu sürümde / In this release

- Android `versionName` ve `versionCode` artık otomatik olarak `package.json` sürümünden üretilir.
- APK sürümü GitHub yayınından önce gerçek paket manifestinden doğrulanır.
- Android yayınları artık aynı kalıcı Ritim sertifikasıyla imzalanır.
- Güncelleme kontrolü 0.7.2 kurulduğunda tekrar indirme önermek yerine uygulamanın güncel olduğunu gösterir.

- Android `versionName` and `versionCode` are now derived automatically from the package version.
- CI verifies the real APK manifest before publishing.
- Android releases now use one persistent Ritim signing certificate.
- The update checker reports 0.7.2 as current instead of offering the same APK again.

> **Bir kerelik kurulum notu:** 0.7.0 ve 0.7.1 farklı geçici sertifikalarla imzalandığı için eski Ritim Android uygulamasını kaldırıp 0.7.2 APK'sını temiz kur. 0.7.3 ve sonraki sürümler doğrudan uygulamanın üstüne kurulabilecek.
>
> **One-time installation note:** Versions 0.7.0 and 0.7.1 used temporary signing certificates. Uninstall the old Android app once, then install 0.7.2 cleanly. Version 0.7.3 and later will install directly over it.
